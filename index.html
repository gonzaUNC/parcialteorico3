Tema 1

Programación Lógica (Prolog)

Motor

Inferencia y Backtracking

Prolog no ejecuta línea por línea. Resuelve objetivos volviendo atrás (backtracking) automáticamente si una rama lógica falla, intentando con la siguiente.

Estructura

Hechos y Reglas

Declaramos verdades absolutas (Hechos) e implicaciones lógicas (Reglas). El motor hace el resto usando Unificación.

Tema 2

Frameworks e Inversión de Control

Un framework provee la arquitectura base. Se rige por la Inversión de Control (IoC): el framework te llama a vos, no vos al framework.

# Código Flask del Parcial
from flask import Flask, request, jsonify
app = Flask(__name__)

def validar_usuario(datos):
    return "nombre" in datos and "email" in datos

@app.route("/usuarios", methods=["POST"])
def crear_usuario():
    datos = request.get_json()
    if not validar_usuario(datos):
        return jsonify({"error": "datos incompletos"}), 400
    return jsonify({"id": 42}), 201
Afirmación del ParcialVerdadero / Falso
`crear_usuario` es un hotspot.Verdadero
El mecanismo de ruteo interno es un frozen spot.Verdadero
El desarrollador invoca a `crear_usuario`.Falso (Lo hace el framework - IoC)
`validar_usuario` es un frozen spot.Falso (Es lógica propia del usuario)
Tema 3

Concurrencia (Sin Actores)

Permite ejecutar tareas en periodos de tiempo superpuestos. Crucial sincronizar para evitar Race Conditions y Deadlocks.

Bajo Nivel

Threads (Hilos)

Mapean directamente a hilos del OS. Tienen estado mutable compartido, requieren sincronización manual (locks, monitores).

Alto Nivel

Futures (Futuros)

Representan un valor que estará disponible de forma asíncrona. Son ideales para componer operaciones sin bloquear el hilo principal.

// Código Scala del Parcial (Solo Threads y Futures)
import scala.concurrent.{Future, Await}
import scala.concurrent.ExecutionContext.Implicits.global
import scala.concurrent.duration._

var contador = 0

// (A) Hilo Nativo
val t = new Thread(() => {
  Thread.sleep(100); contador += 1
})
t.start()

// (B) Futures (Asincronía)
val f1 = Future { Thread.sleep(300); "resultado 1" }
val f2 = Future { Thread.sleep(200); "resultado 2" }
val combinado = Future.sequence(List(f1, f2))

t.join() // Bloqueo manual
val resultados = Await.result(combinado, 5.seconds)
Tema 4

Programación Defensiva y Seguridad

⚠️

El uso de Asserts

Los assert se utilizan para identificar errores lógicos internos durante el desarrollo. No deben usarse en producción para validar input de usuario, ya que pueden ser desactivados globalmente.

# Código de Seguridad del Parcial
def calcular_promedio(valores):
    assert isinstance(valores, list), "valores debe ser una lista"
    assert len(valores) > 0, "la lista no puede estar vacía"
    return sum(valores) / len(valores)

Glosario Bosterizado y Sincronizado

Exclusión Mutua (Mutex)
Mecanismo de concurrencia que asegura que solo un hilo a la vez pueda acceder a una sección crítica o a un recurso compartido, evitando corrupciones.
Sincronización (Sync)
Coordinación de la ejecución de múltiples hilos para asegurar que operen en el orden correcto y no interfieran al acceder a la memoria.
Condición de Carrera (Race Condition)
Fallo que ocurre cuando el resultado de un programa concurrente depende de qué hilo llega primero a modificar una variable compartida.
Deadlock (Abrazo Mortal)
Situación crítica donde dos o más procesos se bloquean eternamente, cada uno esperando que el otro libere un recurso que necesita.
Monitor / Semáforo
Estructuras de alto nivel usadas para implementar exclusión mutua y señalización entre hilos concurrentes.
Inversión de Control (IoC)
El principio de "no nos llames, te llamaremos". El framework toma el flujo de la aplicación y ejecuta tu código cuando corresponde.
Hotspot
Punto específico que expone el framework para que el programador "enchufe" su código personalizado (Ej: @app.route).
Frozen Spot
Código intocable del framework. La arquitectura interna base que ya está resuelta y lista para usarse.
Unificación y Backtracking
En Prolog, el proceso de igualar variables lógicas (Unificación) y la búsqueda hacia atrás probando alternativas si una falla (Backtracking).
