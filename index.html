Tema 1

Programación Lógica (Prolog)

Prolog utiliza un motor de Inferencia que hace Backtracking automático para resolver los subobjetivos.

Ejercicio Base del Parcial

Le preguntamos al intérprete si equipo(elena, ana). es True basándonos en el siguiente código.

% Hechos y Reglas (Del Parcial)
especialista(elena, traumatologia).
departamento(elena, ana).

equipo(X, Y) :- departamento(X, Y).
equipo(X, Y) :- departamento(Y, X).
equipo(X, Y) :- deriva(X, Y).
Subobjetivo para equipo(elena, ana)Evaluación Backtracking
especialista(elena,traumatologia).No se comprueba (no es parte de la regla 'equipo')
departamento(elena,ana).True (Es el primer hecho que verifica la regla)
departamento(ana,elena).No se comprueba (porque el anterior ya dio True, cortando la búsqueda)
deriva(elena,ana).No se comprueba
Tema 2

Frameworks e Inversión de Control

Un framework provee la arquitectura base. Se rige por la Inversión de Control (IoC): el framework llama a tu código mediante los Hotspots.

# Código Flask del Parcial
from flask import Flask, request, jsonify
app = Flask(__name__)

def validar_usuario(datos):
    return "nombre" in datos and "email" in datos

def guardar_en_base(datos):
    return 42

@app.route("/usuarios", methods=["POST"])
def crear_usuario():
    datos = request.get_json()
    if not validar_usuario(datos):
        return jsonify({"error": "datos incompletos"}), 400
    id_nuevo = guardar_en_base(datos)
    return jsonify({"id": id_nuevo}), 201
Afirmación del ParcialVerdadero / Falso
crear_usuario es un hotspot del framework.Verdadero
El mecanismo de ruteo de Flask es un frozen spot.Verdadero
El desarrollador invoca explícitamente a crear_usuario.Falso (Inversión de control)
Flask decide cuándo llamar a crear_usuario.Verdadero
validar_usuario y guardar_en_base son frozen spots.Falso (Son lógica pura del usuario)
Tema 3

Concurrencia

La concurrencia permite ejecutar múltiples tareas de forma simultánea. En Scala podemos manejarla con Hilos (Threads) o con Futuros (Futures).

// Código Scala del Parcial
import scala.concurrent.{Future, Await}
import scala.concurrent.ExecutionContext.Implicits.global
import scala.concurrent.duration._

var contador = 0

// (A) Hilo Nativo
val t = new Thread(() => {
  Thread.sleep(100)
  contador += 1
})
t.start()

// (B) Futures (Alto Nivel)
val f1 = Future { Thread.sleep(300); "resultado 1" }
val f2 = Future { Thread.sleep(200); "resultado 2" }

// (C) Composición
val combinado = Future.sequence(List(f1, f2))

// (D) Bloqueo y Espera
t.join()
val resultados = Await.result(combinado, 5.seconds)
Tema 4

Programación Defensiva y Seguridad

La programación defensiva anticipa fallos. Se utilizan Precondiciones y Postcondiciones para validar que el programa funcione dentro de lo esperado.

# Código de Seguridad del Parcial
def calcular_promedio(valores):
    assert isinstance(valores, list), "valores debe ser una lista"
    assert len(valores) > 0, "la lista no puede estar vacía"
    assert all(isinstance(v, (int, float)) for v in valores), \
        "todos los valores deben ser numéricos"
    return sum(valores) / len(valores)

Atención con los Asserts

Los assert se utilizan para identificar errores lógicos del desarrollador durante la fase de pruebas. No deben usarse para validar inputs de los usuarios en producción, ya que en Python pueden desactivarse globalmente ejecutando el script con la bandera -O.

Glosario Bosterizado

Inversión de Control (IoC)
El framework toma el control del flujo de la aplicación y llama a tu código cuando lo necesita.
Hotspot
Lugar diseñado por el framework para que enchufes tu código personalizado (Ej: un decorador @app.route).
Frozen Spot
El núcleo del framework que no podés tocar, la arquitectura ya resuelta.
Unificación
En Prolog, el motor intenta hacer coincidir dos términos lógicos dándole valor a las variables.
Backtracking
Prolog "vuelve sobre sus pasos" si una rama lógica falla, probando la siguiente alternativa.
Race Condition
Fallo que ocurre cuando el resultado de la ejecución depende del orden impredecible en que se ejecutan los hilos.
